# おしかけログ リリース前チェックリスト

MVPリリース前に手動で確認する項目一覧です。

## 1. 基本動作確認

### ランディングページ
- [ ] トップページが正常に表示される
- [ ] ヒーローセクションのコピー「あの試合、いくら使った？全部わかる。」が表示される
- [ ] 画像が全て正常に表示される（lp-hero.png等）
- [ ] favicon、OGP画像が正しく設定されている
- [ ] 「今すぐはじめる」ボタンが動作する

### マッチログ（/matches）
- [ ] 試合一覧が表示される
- [ ] 「公式から取得」ボタンで最新試合を取得できる
- [ ] 今後の予定と過去の試合が分かれて表示される
- [ ] HOME/AWAYバッジが正しく色分けされている
- [ ] 会場名からGoogle Mapsリンクが開ける
- [ ] フィルター機能が動作する

### 試合詳細（/matches/:id）
- [ ] 試合情報が正しく表示される
- [ ] 費用入力フォームが表示される
- [ ] 交通費・チケット代・飲食代・その他が入力できる
- [ ] 保存ボタンで費用が保存される
- [ ] 保存成功時にトーストが表示される
- [ ] 合計金額が正しく計算される

### 集計（/stats）
- [ ] 年セレクターで年を切り替えられる
- [ ] 観戦試合数が正しく表示される
- [ ] 費用合計・平均が正しく表示される
- [ ] 戦績（勝/分/敗/未確定）が正しく表示される
- [ ] 勝率・不敗率が表示される
- [ ] 0件時に空表示が出る

## 2. モバイル表示確認

以下のページをモバイル表示（幅375px程度）で確認：
- [ ] ランディングページ
- [ ] /matches
- [ ] /matches/:id
- [ ] /stats

確認項目：
- [ ] 横スクロールが発生しない
- [ ] テキストが読める大きさ
- [ ] ボタンがタップしやすい
- [ ] フォーム入力が使いやすい

## 3. エラーハンドリング確認

- [ ] API失敗時にエラーメッセージが表示される
- [ ] 再試行ボタンが機能する
- [ ] ローディング中はスピナーが表示される

## 4. データ整合性

- [ ] 観戦記録がDBに保存される
- [ ] 費用内訳がDBに保存される
- [ ] 集計ページの数値がDBと一致する

## 5. セキュリティ

- [ ] 認証なしでも基本機能が使える（MVP仕様）
- [ ] 環境変数が公開されていない
- [ ] エラーメッセージに内部情報が含まれていない

## 6. パフォーマンス

- [ ] ページ読み込みが5秒以内
- [ ] 試合取得が10秒以内

## 7. 最終確認

- [ ] 全ページでコンソールエラーがない
- [ ] 本番環境で動作確認
- [ ] replit.mdが最新状態
